# First-Pass Plan: 43-9061-office-clerks-general

## Goal
Produce a concise internal records-retention audit report that lets Leo Faraday decide what can be destroyed now, what must be archived, and what requires management review before any action.

## Working assumptions
- The report should separate definite actions from unresolved questions.
- No recommendation should destroy records whose retention status depends on unknown account status or unidentified contents.
- Severity needs to map cleanly to risk: PII/regulatory exposure, policy violation, housekeeping.

## First-pass execution sequence
1. Build a cabinet-by-cabinet inventory table from the raw tally.
2. Classify each item into one of four buckets.
   - Destroy now.
   - Archive offsite.
   - Retain on-site for now.
   - Open question needing Ops/HR/Symitar review.
3. Draft the findings table with severity and target date.
4. Write the disposal queue using explicit retention-schedule citations.
5. Write the archive queue for board packets and older security reports.
6. Add the environmental/security observations as a separate section.

## What must be true in v1
- The exec summary is scannable and counts findings by severity.
- The signature cards are held pending Symitar cross-check.
- The HR boxes stay unresolved pending controlled review.
- The exposed SSNs are elevated and given same-day remediation.

## Main risks
- Recommending destruction based on age alone when retention is conditional.
- Collapsing all findings into the same severity.
- Burying actionable queues inside prose instead of explicit lists.
- Failing to separate clerk-executable actions from manager decisions.

## Immediate iteration queue
- Tighten severity calibration across every finding.
- Quantify lineal feet, binders, and folder counts more consistently.
- Add a more explicit target-date logic by finding type.
- Clarify unsorted timesheets handling and shred vendor scoping.
